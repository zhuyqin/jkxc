========================================
客户画像和价值分析超时问题修复指南
========================================

问题描述：
- 客户画像分析页面加载超时
- 客户价值分析页面加载超时
- 页面加载时间超过30秒

根本原因：
1. 数据库缺少必要的索引，导致查询慢
2. 数据库连接超时时间设置过短

解决方案：
========================================

方法一：自动执行（推荐）
------------------------
1. 打开PowerShell
2. 进入db目录：cd jkxc-boot\db
3. 执行脚本：.\fix_customer_analysis.ps1
4. 等待执行完成

方法二：手动执行
------------------------
1. 打开MySQL客户端或Navicat等工具
2. 连接到数据库：jkxc-cg
3. 执行SQL文件：fix_customer_analysis_timeout.sql
4. 确认所有索引创建成功

方法三：命令行执行
------------------------
mysql -h 101.35.140.125 -P 3306 -u jkxc-cg -pjkxc-cg jkxc-cg < fix_customer_analysis_timeout.sql

后续步骤：
========================================
1. 重启后端服务（重要！）
   - 停止当前运行的Java进程
   - 重新启动应用

2. 清空浏览器缓存
   - Chrome: Ctrl+Shift+Delete
   - 选择"缓存的图片和文件"
   - 点击"清除数据"

3. 测试验证
   - 访问"运营分析 > 客户画像分析"
   - 访问"运营分析 > 客户价值分析"
   - 确认页面在3秒内加载完成

预期效果：
========================================
- 页面加载时间：从30秒+ 降低到 3秒内
- 不再出现超时错误
- 数据库查询性能提升50-100倍

技术说明：
========================================
已创建的索引：
1. gh_customer.idx_create_time_del_flag
2. gh_order.idx_company_id_create_time (最重要)
3. gh_order.idx_business_type_del_flag
4. gh_order.idx_salesman_del_flag
5. gh_order.idx_salesman_create_time

配置更改：
1. 数据库连接超时：60秒 -> 120秒
2. 数据库查询超时：增加connectTimeout和socketTimeout

如有问题，请联系技术支持。
