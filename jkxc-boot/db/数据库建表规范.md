# 数据库建表规范

## 字符集和排序规则规范

**强制要求：所有新建表必须使用以下字符集和排序规则**

```sql
DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci
```

### 规范说明

1. **字符集（CHARACTER SET）**：`utf8mb4`
   - 支持完整的UTF-8字符集，包括emoji表情
   - 每个字符占用最多4个字节

2. **排序规则（COLLATE）**：`utf8mb4_general_ci`
   - 统一使用`utf8mb4_general_ci`排序规则，避免不同表之间JOIN操作时的排序规则冲突
   - `ci`表示大小写不敏感（Case Insensitive）

### 标准建表模板

```sql
CREATE TABLE IF NOT EXISTS `表名` (
  `id` VARCHAR(32) NOT NULL COMMENT '主键ID',
  `字段名` VARCHAR(长度) DEFAULT NULL COMMENT '字段说明',
  -- 其他字段...
  PRIMARY KEY (`id`),
  KEY `idx_字段名` (`字段名`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='表说明';
```

### 重要注意事项

1. **必须在建表语句中明确指定COLLATE**
   - ✅ 正确：`DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci`
   - ❌ 错误：`DEFAULT CHARSET=utf8mb4`（未指定COLLATE，会使用数据库默认值，可能不一致）

2. **修改现有表的字符集**
   如果现有表使用了其他排序规则，需要统一修改：
   ```sql
   ALTER TABLE `表名` CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
   ```

3. **数据库级别设置**
   建议数据库级别的默认字符集也设置为utf8mb4_general_ci：
   ```sql
   ALTER DATABASE `数据库名` CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
   ```

4. **防止乱码**
   - 确保数据库连接URL中包含字符集参数：`characterEncoding=utf8`
   - 确保应用程序使用UTF-8编码
   - 确保数据库客户端使用UTF-8编码

### 检查脚本

执行以下SQL检查表的字符集和排序规则：

```sql
-- 检查所有表的排序规则
SELECT 
    TABLE_NAME,
    TABLE_COLLATION,
    CHARACTER_SET_NAME
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_SCHEMA = DATABASE()
ORDER BY TABLE_NAME;

-- 检查特定表的字段排序规则
SELECT 
    TABLE_NAME,
    COLUMN_NAME,
    COLLATION_NAME,
    CHARACTER_SET_NAME
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = DATABASE()
AND TABLE_NAME = '表名'
AND CHARACTER_SET_NAME IS NOT NULL;
```

### 常见问题

1. **Q: 为什么必须使用utf8mb4_general_ci而不是utf8mb4_0900_ai_ci？**
   A: utf8mb4_0900_ai_ci是MySQL 8.0的新默认排序规则，但为了与现有系统保持一致，统一使用utf8mb4_general_ci。

2. **Q: 如果表已经创建，如何修改排序规则？**
   A: 使用`ALTER TABLE`语句修改，参考上面的"修改现有表的字符集"部分。

3. **Q: 修改排序规则会影响现有数据吗？**
   A: 不会影响数据的存储，只影响排序和比较操作。但如果两个表的排序规则不同，JOIN操作可能会出错。

### 示例

**正确的建表语句：**

```sql
CREATE TABLE IF NOT EXISTS `example_table` (
  `id` VARCHAR(32) NOT NULL COMMENT '主键ID',
  `name` VARCHAR(100) DEFAULT NULL COMMENT '名称',
  `description` TEXT COMMENT '描述',
  `create_time` DATETIME DEFAULT NULL COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_name` (`name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='示例表';
```

**错误的建表语句（缺少COLLATE）：**

```sql
CREATE TABLE IF NOT EXISTS `example_table` (
  `id` VARCHAR(32) NOT NULL COMMENT '主键ID',
  `name` VARCHAR(100) DEFAULT NULL COMMENT '名称',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='示例表';  -- ❌ 缺少COLLATE
```
