# 客户名称同步更新解决方案（完整版）

## 问题描述

在客户管理中修改客户名字（`gh_customer.corporate_name`）后，其他关联表中的公司名称（`company_name`）字段没有同步更新，导致数据不一致。例如：
- 在客户管理中将"ABC公司"改为"ABC科技有限公司"
- 但订单中心、代账合同等模块中仍显示"ABC公司"
- 导致数据查询、统计、报表等功能出现问题

## 涉及的数据表

根据数据库分析，以下表包含需要同步更新的 `company_name` 或 `corporate_name` 字段：

| 序号 | 表名 | 字段名 | 关联方式 | 说明 |
|------|------|--------|----------|------|
| 1 | `gh_customer` | `corporate_name` | - | 主表，客户公司名称 |
| 2 | `gh_order` | `company_name` | `company_id` → `gh_customer.id` | 订单表 |
| 3 | `gh_accounting_contract` | `company_name` | `order_id` → `gh_order.id` → `company_id` | 代账合同表 |
| 4 | `gh_accounting_handover` | `company_name` | `order_id` → `gh_order.id` → `company_id` | 代账交接表 |
| 5 | `gh_business_task` | `company_name` | `order_id` → `gh_order.id` → `company_id` | 业务任务表 |
| 6 | `gh_address_center` | `company_name` | `data_id` → `gh_customer.id` | 地址中心表 |
| 7 | `gh_address_renew` | `company_name` | `company_id` → `gh_customer.id` | 地址续费表 |
| 8 | `gh_bank_diary` | `company_name` | `company_id` → `gh_customer.id` | 银行日记账表 |
| 9 | `gh_reimbursement` | `company_name` | `company_id` → `gh_customer.id` | 报销表 |
| 10 | `gh_opportunity` | `corporate_name` | 通过 `gh_customer_relation` 关联 | 商机表 |

**注意：** `gh_supplier` 表虽然也有 `company_name` 字段，但该字段存储的是供应商自己的公司名称，不是客户公司名称，因此不需要同步更新。

## 解决方案

### 方案一：数据库触发器（推荐）

**优点：**
- 自动执行，无需修改业务代码
- 保证数据一致性
- 对所有更新操作生效（包括直接SQL更新）

**缺点：**
- 可能影响性能（如果关联数据量大）
- 调试相对困难

**实施步骤：**

1. 执行 `sync_customer_name_update.sql` 脚本中的"第一部分"创建触发器
2. 执行"第二部分"修复历史数据
3. 执行"第三部分"验证数据一致性

### 方案二：Java代码层面实现

**优点：**
- 更灵活，可以添加业务逻辑
- 便于调试和维护
- 可以记录详细日志

**缺点：**
- 需要修改业务代码
- 如果有直接SQL更新，可能遗漏

**实施步骤：**

#### 1. 修改 `GhCustomerServiceImpl.java`

在客户更新方法中添加同步逻辑：

```java
@Override
@Transactional(rollbackFor = Exception.class)
public boolean updateById(GhCustomer entity) {
    // 获取更新前的客户信息
    GhCustomer oldCustomer = this.getById(entity.getId());
    
    // 执行客户信息更新
    boolean result = super.updateById(entity);
    
    // 如果客户名称发生变化，同步更新关联表
    if (result && oldCustomer != null 
        && !oldCustomer.getCorporateName().equals(entity.getCorporateName())) {
        syncCustomerNameToRelatedTables(entity.getId(), entity.getCorporateName());
    }
    
    return result;
}

/**
 * 同步客户名称到关联表
 * @param customerId 客户ID
 * @param newCorporateName 新的公司名称
 */
private void syncCustomerNameToRelatedTables(String customerId, String newCorporateName) {
    try {
        // 1. 更新订单表
        ghOrderMapper.updateCompanyNameByCustomerId(customerId, newCorporateName);
        
        // 2. 更新代账合同表（通过订单关联）
        ghAccountingContractMapper.updateCompanyNameByCustomerId(customerId, newCorporateName);
        
        // 3. 更新代账交接表
        ghAccountingHandoverMapper.updateCompanyNameByCustomerId(customerId, newCorporateName);
        
        // 4. 更新业务任务表
        ghBusinessTaskMapper.updateCompanyNameByCustomerId(customerId, newCorporateName);
        
        // 5. 更新地址中心表
        ghAddressCenterMapper.updateCompanyNameByCustomerId(customerId, newCorporateName);
        
        // 6. 更新地址续费表
        ghAddressRenewMapper.updateCompanyNameByCustomerId(customerId, newCorporateName);
        
        // 7. 更新银行日记账表
        ghBankDiaryMapper.updateCompanyNameByCustomerId(customerId, newCorporateName);
        
        // 8. 更新报销表
        ghReimbursementMapper.updateCompanyNameByCustomerId(customerId, newCorporateName);
        
        // 9. 更新商机表
        ghOpportunityMapper.updateCorporateNameByCustomerId(customerId, newCorporateName);
        
        log.info("客户名称同步更新成功: customerId={}, newName={}", customerId, newCorporateName);
    } catch (Exception e) {
        log.error("客户名称同步更新失败: customerId={}, newName={}", customerId, newCorporateName, e);
        throw new RuntimeException("客户名称同步更新失败", e);
    }
}
```

#### 2. 在各个 Mapper 中添加更新方法

以 `GhOrderMapper.java` 为例：

```java
/**
 * 根据客户ID更新公司名称
 * @param customerId 客户ID
 * @param companyName 新的公司名称
 * @return 更新记录数
 */
@Update("UPDATE gh_order SET company_name = #{companyName}, update_time = NOW() " +
        "WHERE company_id = #{customerId} AND del_flag = 0")
int updateCompanyNameByCustomerId(@Param("customerId") String customerId, 
                                   @Param("companyName") String companyName);
```

其他 Mapper 类似实现。

### 方案三：混合方案（最佳实践）

结合触发器和Java代码：

1. **使用触发器**：保证数据库层面的一致性，防止直接SQL操作导致的数据不一致
2. **Java代码记录日志**：在Service层添加日志记录，便于追踪和审计
3. **定期数据校验**：添加定时任务，定期检查数据一致性

```java
@Scheduled(cron = "0 0 2 * * ?") // 每天凌晨2点执行
public void checkCustomerNameConsistency() {
    // 执行数据一致性检查SQL
    List<Map<String, Object>> inconsistentData = 
        ghCustomerMapper.checkNameConsistency();
    
    if (!inconsistentData.isEmpty()) {
        log.warn("发现客户名称不一致的数据: {}", inconsistentData);
        // 发送告警通知
        sendAlertNotification(inconsistentData);
    }
}
```

## 性能优化建议

1. **索引优化**：确保关联字段都有索引
   - `gh_order.company_id`
   - `gh_address_center.data_id`
   - `gh_bank_diary.company_id`
   - 等等

2. **批量更新**：如果需要批量修改客户名称，建议：
   - 关闭触发器
   - 执行批量更新
   - 手动执行同步脚本
   - 重新启用触发器

3. **异步处理**：对于大数据量场景，可以考虑异步更新：
   ```java
   @Async
   public void syncCustomerNameAsync(String customerId, String newName) {
       // 异步执行同步逻辑
   }
   ```

## 常见问题

### Q1: 触发器会影响性能吗？

A: 会有一定影响，但通常可以接受。如果客户关联的订单、合同等数据量特别大（如单个客户有上千条订单），可能会有明显延迟。建议：
- 在测试环境先测试性能
- 考虑异步更新方案
- 优化相关表的索引

### Q2: 如果只想更新部分表怎么办？

A: 可以修改触发器或Java代码，注释掉不需要更新的表。

### Q3: 历史数据修复需要多长时间？

A: 取决于数据量。建议先在测试环境执行，评估时间。通常几万条数据在几秒到几十秒内完成。

### Q4: 触发器和Java代码会不会重复更新？

A: 不会。触发器在数据库层面执行，Java代码如果也实现了同步逻辑，会执行两次。建议：
- 使用触发器方案，不在Java代码中实现
- 或者只在Java代码中实现，不使用触发器

## 总结

**推荐方案：数据库触发器 + 定期数据校验**

1. 执行 `sync_customer_name_update.sql` 创建触发器
2. 执行历史数据修复脚本
3. 添加定时任务定期检查数据一致性
4. 在Java代码中添加日志记录（可选）

这样既保证了数据一致性，又便于维护和监控。
