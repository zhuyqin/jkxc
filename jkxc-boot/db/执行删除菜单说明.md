# 删除菜单SQL执行说明

## 执行前必读
⚠️ **执行前请务必备份数据库！**

## 需要删除的菜单
1. 租户管理
2. 部门管理
3. 我的部门
4. 职务管理
5. 在线开发（及其所有子菜单）
6. 报表设计（及其所有子菜单）
7. 其他使用在线开发组件的菜单

## 执行方式

### 方式一：使用MySQL命令行执行（推荐）

```bash
# Windows PowerShell
mysql -h localhost -u root -proot jeecg-boot < db/delete_menus.sql

# 或者先登录MySQL，然后执行
mysql -h localhost -u root -proot jeecg-boot
source db/delete_menus.sql;
```

### 方式二：使用数据库管理工具执行

1. 使用 Navicat、DBeaver、phpMyAdmin 等工具连接数据库
2. 打开 `jkxc-boot/db/delete_menus.sql` 文件
3. 执行整个SQL脚本

### 方式三：逐条执行（如果遇到外键约束问题）

如果遇到外键约束问题，可以按照以下顺序执行：

1. 先删除角色权限关联表中的记录
2. 再删除菜单数据规则表中的记录
3. 最后删除菜单表中的记录

## 验证删除结果

执行完成后，可以运行以下SQL验证：

```sql
-- 检查是否还有相关菜单
SELECT id, name, component, url FROM sys_permission 
WHERE id IN (
    '1280350452934307841', -- 租户管理
    '45c966826eeff4c99b8f8ebfe74511fc', -- 部门管理
    '5c2f42277948043026b7a14692456828', -- 我的部门
    '1174506953255182338', -- 职务管理
    'e41b69c57a941a3bbcce45032fe57605', -- 在线开发
    '1205097455226462210' -- 报表设计
);

-- 应该返回空结果
```

## 注意事项

1. 执行前请确保已停止应用服务
2. 建议在测试环境先执行验证
3. 如果菜单被其他功能引用，可能需要额外处理
4. 删除后需要重启应用服务才能生效
