# 首页设计方案

## 一、设计思路

### 1. 两个首页视图

#### 1.1 数据统计页（Dashboard）- 给领导看
**目标用户**：管理层、决策者
**核心内容**：
- 关键业务指标（KPI卡片）
- 业务趋势图表
- 数据概览和对比
- 快速跳转到详细分析页面

**展示内容**：
- 今日/本月/本年订单数、金额
- 客户数量统计
- 任务完成情况
- 业务类型分布
- 业务员业绩排名
- 月度/年度趋势图表
- 关键指标对比（同比、环比）

#### 1.2 工作台（Workbench）- 给员工看
**目标用户**：普通员工、业务员
**核心内容**：
- 待办任务列表
- 快捷操作入口
- 我的工作统计
- 消息通知

**展示内容**：
- 待处理任务（工商任务、审核任务等）
- 待处理订单
- 我的客户
- 快捷入口（创建订单、添加客户等）
- 我的业绩统计
- 系统通知

### 2. 实现方案

#### 方案A：基于角色自动跳转（推荐）
- 通过 `RoleIndexConfigEnum` 配置角色与首页的映射
- 登录后根据用户角色自动跳转到对应首页
- 优点：用户体验好，无需手动切换
- 缺点：需要维护角色配置

#### 方案B：首页切换器
- 在首页提供切换按钮，用户可以手动切换
- 优点：灵活，用户可以自由选择
- 缺点：需要用户手动操作

#### 方案C：混合方案（最佳）
- 默认根据角色自动跳转
- 提供切换按钮，允许用户手动切换
- 记住用户的选择偏好

## 二、技术实现

### 1. 后端接口

#### 1.1 数据统计页接口
```
GET /dashboard/getStatistics
返回：
- 订单统计（今日/本月/本年）
- 客户统计
- 任务统计
- 业务类型分布
- 业务员排名
- 趋势数据
```

#### 1.2 工作台接口
```
GET /dashboard/getWorkbench
返回：
- 待办任务列表
- 待处理订单
- 我的客户统计
- 快捷操作列表
- 消息通知
```

### 2. 前端页面

#### 2.1 数据统计页
- 路径：`dashboard/Dashboard.vue`
- 组件：统计卡片、图表、数据表格

#### 2.2 工作台
- 路径：`dashboard/Workbench.vue`
- 组件：待办列表、快捷入口、统计卡片

### 3. 角色配置

在 `RoleIndexConfigEnum` 中添加：
- 领导角色 → `dashboard/Dashboard`
- 员工角色 → `dashboard/Workbench`

## 三、页面布局设计

### 数据统计页布局
```
┌─────────────────────────────────────────┐
│  关键指标卡片（4-6个）                    │
├─────────────────────────────────────────┤
│  趋势图表（月度/年度）                    │
├──────────────────┬──────────────────────┤
│  业务类型分布     │  业务员排名           │
├──────────────────┴──────────────────────┤
│  数据表格（最近订单/客户）                │
└─────────────────────────────────────────┘
```

### 工作台布局
```
┌─────────────────────────────────────────┐
│  欢迎信息 + 我的统计卡片（2-4个）          │
├──────────────────┬──────────────────────┤
│  待办任务列表     │  快捷操作入口         │
├──────────────────┼──────────────────────┤
│  待处理订单       │  系统通知             │
└─────────────────────────────────────────┘
```
