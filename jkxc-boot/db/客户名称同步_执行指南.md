## ⚠️ 常见问题处理

### 问题1：触发器创建失败

**错误信息：** `ERROR 1419: You do not have the SUPER privilege`

**解决方案：**
```sql
-- 临时设置（仅当前会话）
SET GLOBAL log_bin_trust_function_creators = 1;

-- 然后重新执行创建触发器的脚本
```

### 问题2：历史数据修复很慢

**原因：** 数据量大，缺少索引

**解决方案：**
1. 分批执行，每次处理1000条
2. 检查并添加必要的索引
3. 在业务低峰期执行

```sql
-- 分批修复示例（订单表）
UPDATE gh_order o
INNER JOIN gh_customer c ON o.company_id = c.id
SET o.company_name = c.corporate_name,
    o.update_time = NOW(),
    o.update_by = 'system_sync'
WHERE o.company_name != c.corporate_name
  AND o.del_flag = 0
  AND c.del_flag = 0
LIMIT 1000;

-- 重复执行直到影响行数为0
```

### 问题3：触发器影响性能

**症状：** 修改客户名称时响应很慢

**解决方案：**
1. 检查关联表的数据量
2. 优化索引
3. 考虑改用异步更新（需要修改Java代码）

```sql
-- 检查单个客户的关联数据量
SELECT 
    'gh_order' AS table_name,
    COUNT(*) AS count
FROM gh_order 
WHERE company_id = 'YOUR_CUSTOMER_ID' AND del_flag = 0

UNION ALL

SELECT 
    'gh_accounting_contract' AS table_name,
    COUNT(*) AS count
FROM gh_accounting_contract ac
INNER JOIN gh_order o ON ac.order_id = o.id
WHERE o.company_id = 'YOUR_CUSTOMER_ID' AND ac.del_flag = 0;

-- 如果某个客户的关联数据超过1000条，建议使用异步更新
```

### 问题4：数据仍然不一致

**可能原因：**
1. 触发器未生效
2. 有直接SQL更新绕过了触发器
3. 存在并发更新

**解决方案：**
```sql
-- 1. 检查触发器状态
SHOW TRIGGERS WHERE `Trigger` = 'trg_customer_name_update';

-- 2. 手动修复特定客户
SET @customer_id = 'YOUR_CUSTOMER_ID';
SET @new_name = (SELECT corporate_name FROM gh_customer WHERE id = @customer_id);

-- 执行 customer_name_sync_maintenance.sql 中的第3部分

-- 3. 设置定时任务，每天凌晨自动检查和修复
```
