# 客户双向关联功能说明

## 功能概述

客户管理系统支持客户之间的双向关联功能。当客户A关联客户B时，系统会自动建立B到A的反向关联，确保关联关系的一致性。

## 核心特性

### 1. 双向自动关联
- 当A关联B时，B自动关联A
- 当A取消关联B时，B也自动取消关联A
- 保证关联关系的对称性和一致性

### 2. 批量关联管理
- 支持一次性设置多个关联企业
- 自动计算新增和删除的关联
- 批量更新所有受影响的客户记录

### 3. 事务保证
- 使用批量更新确保操作的原子性
- 避免出现单向关联的不一致状态

## 实现细节

### 后端实现

#### 1. setRelations 方法（批量设置关联）

**位置**: `GhCustomerController.java`

**功能**:
- 设置当前客户的关联企业列表
- 自动建立双向关联关系
- 处理新增和删除的关联

**实现逻辑**:
```java
1. 获取当前客户的旧关联列表（oldRelatedIds）
2. 获取新的关联列表（newRelatedIds）
3. 计算需要新增的关联（toAdd = newRelatedIds - oldRelatedIds）
4. 计算需要删除的关联（toRemove = oldRelatedIds - newRelatedIds）
5. 更新当前客户的关联列表
6. 对于每个新增的关联：
   - 获取关联客户
   - 在其关联列表中添加当前客户ID
7. 对于每个删除的关联：
   - 获取关联客户
   - 从其关联列表中移除当前客户ID
8. 批量更新所有受影响的客户记录
```

**示例**:
```
客户A当前关联: [B, C]
新设置关联: [C, D]

处理过程:
- toAdd = [D]（新增D）
- toRemove = [B]（删除B）
- 更新A的关联为 [C, D]
- 在D的关联中添加A
- 从B的关联中移除A
```

#### 2. removeRelation 方法（移除单个关联）

**位置**: `GhCustomerController.java`

**功能**:
- 移除两个客户之间的关联关系
- 双向移除，确保一致性

**实现逻辑**:
```java
1. 验证两个客户都存在
2. 从客户A的关联列表中移除客户B
3. 从客户B的关联列表中移除客户A
4. 批量更新两个客户记录
```

**示例**:
```
客户A关联: [B, C]
客户B关联: [A, D]

移除A和B的关联后:
客户A关联: [C]
客户B关联: [D]
```

## API 接口

### 1. 批量设置关联企业

**接口**: `POST /customer/ghCustomer/setRelations`

**参数**:
- `customerId` (query): 当前客户ID
- `relatedCompanyIds` (body): 关联企业ID列表（数组）

**返回**:
```json
{
  "success": true,
  "message": "设置成功"
}
```

**示例请求**:
```javascript
POST /customer/ghCustomer/setRelations?customerId=1234567890

## 数据结构

### gh_customer 表

**关联字段**:
- `related_company_id` (VARCHAR): 存储关联企业ID，多个ID用逗号分隔
  - 示例: "1111111111,2222222222,3333333333"
  - 空值表示没有关联企业

**存储格式**:
```
客户A (id=1234567890)
  related_company_id = "1111111111,2222222222"
  
客户B (id=1111111111)
  related_company_id = "1234567890,3333333333"
  
客户C (id=2222222222)
  related_company_id = "1234567890"
```

```

## 注意事项

### 1. 不能关联自己
- 系统会验证关联列表中不能包含当前客户自己
- 如果尝试关联自己，会返回错误提示

### 2. 关联关系的对称性
- 所有关联关系都是双向的
- 不存在单向关联的情况
- 如果发现数据不一致，可能是旧数据，需要手动修复

### 3. 批量更新的原子性
- 使用 `updateBatchById` 方法批量更新
- 确保所有关联关系同时更新成功或失败
- 避免出现部分更新的情况

### 4. 性能考虑
- 关联数量较多时，批量更新可能需要较长时间
- 建议限制单个客户的关联数量（如不超过50个）
- 大量关联操作建议在业务低峰期进行

## 故障排查

### 问题1: 关联关系不对称
**现象**: 客户A关联了B，但B没有关联A

**原因**: 
- 可能是旧数据，在实现双向关联之前创建的
- 或者是并发操作导致的数据不一致

**解决方案**:
```sql
-- 查找不对称的关联关系
SELECT 
    a.id as customer_a_id,
    a.corporate_name as customer_a_name,
    a.related_company_id as a_related,
    b.id as customer_b_id,
    b.corporate_name as customer_b_name,
    b.related_company_id as b_related
FROM gh_customer a
JOIN gh_customer b ON FIND_IN_SET(b.id, a.related_company_id) > 0
WHERE FIND_IN_SET(a.id, b.related_company_id) = 0
  AND a.del_flag = 0
  AND b.del_flag = 0;

-- 修复不对称关系（需要根据实际情况调整）
-- 方案1: 建立双向关联
-- 方案2: 删除单向关联
```

### 问题2: 关联列表中有无效ID
**现象**: related_company_id字段中包含已删除或不存在的客户ID

**解决方案**:
```sql
-- 查找包含无效ID的记录
SELECT 
    c.id,
    c.corporate_name,
    c.related_company_id
FROM gh_customer c
WHERE c.del_flag = 0
  AND c.related_company_id IS NOT NULL
  AND c.related_company_id != ''
  AND EXISTS (
    SELECT 1
    FROM (
      SELECT SUBSTRING_INDEX(SUBSTRING_INDEX(c.related_company_id, ',', numbers.n), ',', -1) as related_id
      FROM (SELECT 1 n UNION SELECT 2 UNION SELECT 3 UNION SELECT 4 UNION SELECT 5) numbers
      WHERE CHAR_LENGTH(c.related_company_id) - CHAR_LENGTH(REPLACE(c.related_company_id, ',', '')) >= numbers.n - 1
    ) ids
    WHERE NOT EXISTS (
      SELECT 1 FROM gh_customer c2 
      WHERE c2.id = ids.related_id AND c2.del_flag = 0
    )
  );

-- 清理无效ID（需要编写清理脚本）
```
